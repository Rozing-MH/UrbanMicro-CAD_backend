### 2.1 数据库约束

- Supabase 数据库已存在，首版后端不执行 DDL。
- 如果后续发现字段、索引或 RLS 策略需要调整，必须先单独确认，再通过受控迁移执行。
- `topology_data`、`rule_data`、`snapshot_data` 为 JSONB，是前端路网、交规、OD 与快照数据的主要持久化载体。
- `bbox` 为 PostGIS geometry 字段，首版可保存为空，后续再按项目范围计算写入。

## 3. 实际数据库结构

### 3.1 `sys_user`

用途：系统用户与鉴权主体。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT identity | 主键。 |
| `username` | varchar | 唯一用户名，长度约束不小于 3。 |
| `password_hash` | varchar | 密码哈希。 |
| `role` | `enum_user_role` | 默认 `USER`。 |
| `created_at` / `updated_at` | timestamptz | 审计时间。 |
| `is_deleted` | boolean | 软删除。 |

后端要求：

- 登录时只返回 JWT 和用户概要，不返回 `password_hash`。
- 注册/创建用户时必须使用 BCrypt 或 Argon2 哈希密码。
- 所有项目、快照、报表查询必须按当前用户隔离。

### 3.2 `prj_project`

用途：工程基础信息与当前快照数据。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | UUID | 工程 ID。 |
| `user_id` | BIGINT | 归属用户，关联 `sys_user.id`。 |
| `name` | varchar | 工程名，非空。 |
| `description` | text | 工程说明。 |
| `bbox` | geometry | 工程空间范围，首版可为空。 |
| `topology_data` | jsonb | 当前路网拓扑数据。 |
| `rule_data` | jsonb | 当前交规、OD、信控等规则数据。 |
| `version` | int | 当前版本号。 |
| `created_at` / `updated_at` | timestamptz | 审计时间。 |
| `is_deleted` | boolean | 软删除。 |

后端要求：

- 项目列表默认过滤 `is_deleted = false` 和当前用户。
- 保存快照时同步更新 `topology_data`、`rule_data`、`description` 与 `version`。
- 删除项目采用软删除，并确保列表与详情接口不可再读取。

### 3.3 `prj_snapshot`

用途：工程历史快照。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | UUID | 快照 ID。 |
| `project_id` | UUID | 关联工程，工程删除时级联。 |
| `version` | int | 快照版本。 |
| `snapshot_data` | jsonb | 完整快照数据。 |
| `description` | varchar | 快照说明。 |
| `created_at` / `updated_at` | timestamptz | 审计时间。 |
| `is_deleted` | boolean | 软删除。 |

后端要求：

- `PUT /api/projects/{id}/snapshot` 每次保存创建一条快照记录。
- 快照内容建议统一为：`{ topologyData, ruleData, description, savedAt }`。
- 快照列表和回滚需验证当前用户拥有对应工程。

### 3.4 `prj_template`

用途：样例库与断面/工程模板。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | UUID | 模板 ID。 |
| `name` | varchar | 模板名称。 |
| `category` | `enum_template_category` | 模板分类。 |
| `snapshot_data` | jsonb | 模板快照数据。 |
| `thumbnail_url` | text | 缩略图地址。 |
| `created_at` / `updated_at` | timestamptz | 审计时间。 |
| `is_deleted` | boolean | 软删除。 |

当前分类枚举：

- `BASIC_INTERSECTION`
- `CLOVERLEAF`
- `TURBINE`
- `DIAMOND`
- `ROUNDABOUT`
- `CUSTOM`

后端要求：

- 首版实现只读模板列表与详情。
- 后续实现用户自定义模板保存时，需明确 `CUSTOM` 的权限模型。

### 3.5 `rpt_evaluation_report`

用途：评估报表与仿真结果归档。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | UUID | 报表 ID。 |
| `project_id` | UUID | 关联工程。 |
| `user_id` | BIGINT | 归属用户。 |
| `lane_metrics` | jsonb | 车道指标。 |
| `intersection_los` | jsonb | 交叉口 LOS。 |
| `heatmap_config` | jsonb | 热力图配置。 |
| `flight_line_data` | jsonb | 飞线/轨迹数据。 |
| `generated_at` | timestamptz | 生成时间。 |
| `created_at` / `updated_at` | timestamptz | 审计时间。 |
| `is_deleted` | boolean | 软删除。 |

后端要求：

- 首版 `POST /api/reports/export` 可先生成 CSV/JSON/PDF 占位能力中的一种，并保存元数据。
- 后续补齐 PDF 报表、截图嵌入、LOS 表格和热力图配置归档。

## 5. API 契约计划

### 5.1 统一响应格式

后端统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

前端当前 `isApiSuccess()` 对数字 `code` 使用 HTTP 成功区间判断，因此后端新接口统一使用 `code = 200` 表示成功。

错误建议：

| code | 含义 |
| --- | --- |
| `200` | 成功 |
| `400` | 参数错误 |
| `401` | 未登录或 token 失效 |
| `403` | 无权限 |
| `404` | 资源不存在 |
| `409` | 状态冲突 |
| `500` | 服务器内部错误 |

### 5.2 鉴权接口

| 方法 | 路径 | 用途 | 首版优先级 |
| --- | --- | --- | --- |
| `POST` | `/api/auth/register` | 注册用户 | P1 |
| `POST` | `/api/auth/login` | 登录并返回 JWT | P0 |
| `GET` | `/api/auth/me` | 获取当前用户 | P0 |
| `POST` | `/api/auth/logout` | 登出 | P2 |

`LoginResponse` 已按前端当前契约返回：

```json
{
  "token": "jwt",
  "userId": "1",
  "username": "demo",
  "role": "USER"
}
```

### 5.3 工程接口

| 方法 | 路径 | 用途 | 首版优先级 |
| --- | --- | --- | --- |
| `GET` | `/api/projects` | 当前用户工程列表 | P0 |
| `POST` | `/api/projects` | 创建工程 | P0 |
| `GET` | `/api/projects/{id}` | 打开工程详情 | P0 |
| `PUT` | `/api/projects/{id}` | 更新工程基础信息 | P1 |
| `DELETE` | `/api/projects/{id}` | 软删除工程 | P1 |
| `PUT` | `/api/projects/{id}/snapshot` | 保存当前工程快照 | P0 |
| `GET` | `/api/projects/{id}/snapshots` | 快照列表 | P1 |
| `POST` | `/api/projects/{id}/snapshots/{snapshotId}/restore` | 回滚到快照 | P1 |

前端当前保存请求体：

```json
{
  "topologyData": {},
  "ruleData": {},
  "description": "保存说明"
}
```

工程详情响应中的关键字段必须包含：

```json
{
  "id": "uuid",
  "name": "项目名",
  "description": "说明",
  "topologyData": {},
  "ruleData": {},
  "version": 1,
  "createdAt": "2026-05-27T00:00:00Z",
  "updatedAt": "2026-05-27T00:00:00Z"
}
```

### 5.4 模板接口

| 方法 | 路径 | 用途 | 首版优先级 |
| --- | --- | --- | --- |
| `GET` | `/api/templates` | 模板列表，支持 category 过滤 | P0 |
| `GET` | `/api/templates/{id}` | 模板详情 | P0 |
| `POST` | `/api/templates` | 保存用户自定义模板 | P2 |
| `DELETE` | `/api/templates/{id}` | 删除用户自定义模板 | P2 |

模板列表响应建议包含：

```json
{
  "id": "uuid",
  "name": "基础十字路口",
  "category": "BASIC_INTERSECTION",
  "thumbnailUrl": "",
  "createdAt": "2026-05-27T00:00:00Z"
}
```

模板详情额外包含 `snapshotData`。

### 5.5 报表接口

| 方法 | 路径 | 用途 | 首版优先级 |
| --- | --- | --- | --- |
| `POST` | `/api/reports/generate` | 生成并保存报表摘要 | P1 |
| `GET` | `/api/reports?projectId=...` | 工程报表列表 | P1 |
| `POST` | `/api/reports/export` | 导出 CSV 报表 | P1 |
| `GET` | `/api/reports/{id}/download` | 下载已有报表 CSV | P1 |
| `GET` | `/api/reports/{id}` | 报表详情 | P2 |

首版导出请求体建议：

```json
{
  "projectId": "uuid",

  "laneMetrics": {},
```

## 9. 风险与处理策略

| 风险 | 说明 | 策略 |
| --- | --- | --- |
| Supabase RLS 与后端鉴权重复 | RLS 已开启，后端也会做用户隔离，可能出现策略不匹配。 | 首版以后端应用层权限为主，联调时逐项确认 RLS 不阻断服务账号访问。 |
| JSONB 契约演进 | 前端 `TopologyData` / `RuleData` 会继续扩展。 | 后端不强行展开业务字段，只校验必要外层结构，避免频繁迁移。 |
| `bbox` 几何字段首版未使用 | 前端当前主要保存 JSONB，空间范围后续才需要检索。 | 首版允许为空，后续按拓扑节点计算 PostGIS bbox。 |
| 报表导出范围较大 | PDF、截图、热力图和 LOS 表需要多模块协同。 | 首版先做 CSV/JSON 或最小 PDF，后续迭代完整报表。 |
| 后端目录为空 | 已完成首轮脚手架与核心接口。 | 后续重点转向真实数据库联调、集成测试和部署配置。 |
| 请求体与 JSON 安全边界 | 已增加 `/api` 请求体大小限制、Jackson 解析约束和服务层 JSON 校验。 | 后续按前端真实数据规模调整阈值，并补 JSON schema/typed DTO 校验。 |
| 登录注册滥用 | 代码内暂未实现限流和锁定策略。 | 部署网关限流或后续引入 Bucket4j/Redis。 |
